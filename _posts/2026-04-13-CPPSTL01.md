## 课前复习

```cpp
//诡异操作讨论
#include <iostream>
using namespace std;
class A   
{
public: virtual void Func() { cout << "A::Func" << endl; } 
};
class B:public A  {
public: virtual void Func() { cout << "B::Func" << endl; } 
};
int main()  {
A a;
A * pa = new B();
pa->Func();
long * p1 = (long * ) & a; // p1指向a的第一个8字节 (虚指针vptr_A)
////若是64位程序, 指针为8字节。在64位windows下应改为long long（8字节）。（64位linux/unix下long也是八字节）

long * p2 = (long * ) pa;//p2指向B对象的第一个8字节 (vptr_B)

* p2 = * p1;// 将对象 a 的虚指针拷贝给 pa 指向的对象，强行将其 vptr 修改为 vptr_A
pa->Func();//通过vptr_A找到A的虚函数表, 调用A::Func 
return 0;  
}
//对象 a 的内存布局中只有 一个 vptr。这个指针指向一个数组（即虚函数表），数组里面按顺序存放着各个虚函数的函数地址。


```


| 对象/变量 | 内存起始位置的内容 (vptr) | 指向的虚函数表 (vtable) | 虚表中的内容 (函数地址) | 执行 `pa->Func()` 的逻辑 |
| :--- | :--- | :--- | :--- | :--- |
| 对象 a(A 类型) | 指向类 A 虚函数表指针 `vptr_A` | 类 A 的虚函数表 | `A::Func` 的地址 | - |
| 指针 pa (指向 B 对象) | 指向类 B 虚函数表指针 `vptr_B` | 类 B 的虚函数表 | `B::Func` 的地址 | 通过 `vptr_B` 找到 B 的虚表，调用 `B::Func` |
|篡改后pa（指向对象）| 指向类A虚函数表指针`vptr_A`|类A的虚函数表|`A::func`的地址|-|

### stl基本概念

• 容器: 可容纳各种数据类型的数据结构  

• 迭代器: 可依次存取容器中元素的工具  
（ 普通C++指针就是一种迭代器）  

• 算法: 用来操作容器中元素的函数模板  

 可以用于存放各种类型的数据 (基本类型的变量, 对象
等) 的数据结构


#### 分类：

1) 顺序容器/序列容器  
元素的插入位置同元素值无关  

vector, deque, list  
2) 关联容器/有序容器  
set, multiset, map, multimap 
容器内的元素是排序的,插入任何元素,都按排序准则确定位置  
查找性能非常好   

------以上两种容器称为第一类容器-----

3) 容器适配器  
stack, queue, priority_queu  

• 对象插入容器, 实际插入对象复
制品（复制构造）    
• 许多算法,要求对容器中的元素
进行比较, 所以放入容器的对象所属的类, 还应
该实现 ==和 <运算符  

#### 1.1 顺序容器简介
1) vector 头文件 <vector>
动态数组, 随机存取任何元素都能在常数时间完成, 在尾端增（.pushback）删
性能较好  
2) deque 头文件 <deque>
动态数组, 随机存取任何元素都能在常数时间完成 (次
于vector), 在两端增删性能较好
3) list 头文件 <list>
双向链表, 在任何位置增删元素都能在常数时间完成, 不支持随
机存取  
顺序容器,元素的插入位置同元素的值无关

#### 1.2 关联容器

容器内的元素是排序的,插入任何元素,都按排序准则确定位置  
关联容器在“查找”时性能非常好  
1) set/multiset: 头文件 set
set是集合,不允许相同元素,  
但multiset中允许存在相同的元素  

2) map/multimap: 头文件 map

映射,map与set的不同在于map中存放的是成对的key/value  
并根据key对元素进行排序,可快速根据key来检索元素。  
map同multimap的不同在于是否允许相同key的元素。  
上述四种容器通常以平衡二叉树方式实现，插入和检索的时间都是 O(logN)  

#### 1.3 容器适配器

1) stack: 头文件 stack  
栈. 有限序列,FILO后进先出，只能操作栈顶元素  

2)  queue: 头文件 queue
队列. 尾部插入,删除、检索和修改只允许从头部
进行. 先进先出,FIFO  

3）priority_queue: 头文件 <queue>
优先队列. 最高优先级元素总是第一个出列  

#### stl容器共有成员函数  

1) 所有标准库容器共有的成员函数：
• 相当于按词典顺序比较两个容器的运算符: 
=, < , <= , >,  >=, == , !=  
（从第一个开始每个元素比较）  
• empty:是否有元素    
• max_size: 最多能装多少元素  
• size:  容器中元素个数  
• swap:交换两个容器内容  

####  只在第一类容器中的函数
• begin 返回指向容器中第一个元素的迭代器  
• end 返回指向容器中最后一个元素后面的位置的迭代器  
• rbegin 返回指向容器中最后一个元素的迭代器    
• rend 返回指向容器中第一个元素前面的位置的迭代器  
• erase 从容器中删除一个或几个元素  
• clear 从容器中删除所有元素  

###  迭代器
• 用于指向第一类容器中的元素, 有const和非 const  
• 通过迭代器可以读取它指向的元素  
通过非const迭代器还能修改  
迭代器用法和指针类似  

• 定义一个容器类的迭代器的方法可以是：  
容器类名::iterator   变量名; （或const_iterator） 

• 访问一个迭代器指向的元素：*迭代器变量名  

• 迭代器可以执行 ++ 操作, 以指向下一个元素    

迭代器到达了容器末端元素后面, 变成past-the-end   

 example:

 ```cpp
例：
#include <vector>
#include <iostream>
using namespace std;
int main()  {
vector<int> v;    //一个存放int元素的空向量
v.push_back(1);
v.push_back(2); 
v.push_back(3);  
v.push_back(4);
vector<int>::const_iterator i;   //常量迭代器
for( i = v.begin(); i != v.end(); i ++ ) 
cout << * i << ", ";
cout << endl;
25
vector<int>::reverse_iterator r;  //反向迭代器
for( r = v.rbegin(); r != v.rend(); r++ ) 
cout << * r << ", ";
cout << endl;
vector<int>::iterator j;   //非常量迭代器
for( j = v.begin(); j != v.end(); j ++ ) 
* j =  100;
for( i = v.begin(); i != v.end(); i++ ) 
cout << * i << ", ";
}

输出结果：
1, 2, 3, 4,
4, 3, 2, 1,
100, 100, 100, 100,


 ```
#### 注意
 不同容器支持的迭代器强弱不同  
• 容器的迭代器的功能强弱, 决定了该容器是否支持
STL中的某种算法  
• 只有第一类容器能用迭代器遍历  
• 排序算法需要通过随机迭代器来访问容器中的元素,
那么有的容器就不支持排序算法  


##### STL中的迭代器按功能由弱到强分为5种：  
1. 输入：Input iterators 提供对数据的只读访问    
1. 输出：Output iterators 提供对数据的只写访问   

2. 正向：Forward iterators 提供读写操作, 并能一次一个地向前
推进迭代器    

3. 双向：Bidirectional iterators提供读写操作, 并能一次一个地
向前和向后移动    

4. 随机访问：Random access iterators提供读写操作, 并能在数
据中随机移动    
编号大的迭代器能当作编号小的迭代器使用    

###### 操作功能：
```cpp
•  所有迭代器： ++p, p ++
• 输入迭代器： * p, p = p1, p ==p1, p!= p1
• 输出迭代器： * p, p = p1
• 正向迭代器： 上面全部
•  双向迭代器： 上面全部,--p, p --,  
• 随机访问迭代器： 上面全部, 
以及：{
移动i个单元:      p+= i, p -= i, p + i, p – i
大于/小于比较:  p < p1, p <= p1, p > p1, p>= p1
下标p[i]: p 后面的第i个的引用
}
```

| 容器 | 迭代器类别 |
| :--- | :--- |
| vector | 随机 |
| deque | 随机 |
| list | 双向 |
| --- | --- |
| set/multiset | 双向 |
| map/multimap | 双向 |
| --- | --- |
| stack | 不支持迭代器 |
| queue | 不支持迭代器 |
| priority_queue | 不支持迭代器 |

### 通用stl算法

• 算法就是一个个函数模板  
• 算法通过迭代器来操纵容器中的元素  
• 许多算法需要两个参数,一个是起始元素的迭代器,  
• 一个是终止元素的后面一个元素的迭代器 （如排序和查找）   

• 有的算法返回一个迭代器.例如 find() 算法,在容器中 
查找一个元素,并返回一个指向该元素的迭代器。  
• 算法可以处理容器,也可以处理C风格数组  

#### 分类
1) 变化序列算法:
copy, remove, fill, replace, random_shuffle, swap, …  
会改变容器  
2) 非变化序列算法:   
adjacent-find, equal, mismatch, find, count, search,   
count_if, for_each, search_n  


```
以上函数模板都在 <algorithm> 中定义
还有其他算法,例如 <numeric>中的算法
```

#### 算法示例: find()

```
template<class InIt, class T>
InIt find(InIt first, InIt last, const T& val);
``` 

• first 和 last 都是容器的迭代器,查找区间起点和终点
• 区间左闭右开!!!即区间的起点是位于查找范围,而终点不是  

• val:要查找的值  
• 函数返回一个迭代器  
• 如果找到,指向被找到的元素  
• 如果找不到,指向查找区间终点  

```cpp

#include <vector>
#include <algorithm>
#include <iostream>
using namespace std;
int main() {
int array[10] = {10, 20, 30, 40};
vector<int> v;
v.push_back(1);
v.push_back(3);
v.push_back(2);
v.push_back(4);
vector<int>::iterator p;
p = find(v.begin(), v.end(), 3);
if( p != v.end())
cout << * p << endl;
37
p = find(v.begin(), v.end(), 9);
if( p == v.end())
cout << "not found " << endl;
p = find(v.begin()+1, v.end()-2, 1);    //查找区间[2, 3)
if( p != v.end())
cout << * p << endl;
int * pp = find(array, array+4, 20);
cout << * pp << endl;
}

```

###### 迭代器前是否要加typename：

```cpp
在模板内部，利用“依赖模板参数的类”的成员类型来定义新变量，加上 typename。    
此时因为类型不指定，编译器不知道这是类型还是成员。  

class Student { ... };  // 假设已定义

template <class T>
void PrintList(const list<T>& lst) {
    // 情况1：左边依赖 T，必须加 typename
    typename list<T>::const_iterator it1;  // 必须.list<T>是依赖名
    
    // 情况2：左边是具体类 Student，不需要 typename。Student有个重名变量也无所谓。
    list<Student>::const_iterator it2;     // 完全合法，无需 typename。想写也可以写。
}

```

### 结论

vector: 强调通过随机访问进行快速访问  
插入/删除：非尾处O(n)或结尾处O(1)   

list: 强调元素的快速插入和删除, 不支持随机访问迭
代器    
插入/删除为O(1)  

 deque: 类似vector容器, 但强调在两端处的快速插入
和删除均为O(1)  




## 函数对象
一个类重载了( )为成员函数 →该类为函数对象类  
这个类的对象 →函数对象    
看上去像函数调用, 实际上也执行了函数调用    
比如

```cpp

less<T>：返回 a < b（实现升序）。
greater<T>：返回 a > b（实现降序）。

```

这样有好处。
• 为了可复用，STL子操作是参数化的。（只给出框架。如sort传compare）   
• 例如：sort的排序原则 (顺序/ 逆序)  
• 函数对象就是用来描述这些子操作的对象  

函数对象比如

```cpp

class CMyAverage {
public:
可以在此补充其他比较固定的参数而无需全局变量，更灵活。
double operator() (int a1, int a2, int a3) {
//重载 ( ) 运算符

return (double)(a1 + a2 + a3) / 3;
}
};  //重载 ( ) 运算符时, 参数可以是任意多个

CMyAverage Average;  //函数对象
cout << Average(3, 2, 3);


```


应用

```cpp

STL里有以下模板:
#include <numeric>//在这里头
template<class InIt, class T, class Pred> 
T accumulate(InIt first, InIt last, T val, Pred pr); 

val：初始值和最终累积到的容器。
pr：一个函数对象或函数或函数指针。（如何累计）  

对[first,  last)中的每个迭代器I指向的对象, 执行 val= pr(val,*I), 并返回最终的val
```
