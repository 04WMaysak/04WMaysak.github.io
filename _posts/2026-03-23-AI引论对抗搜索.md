应对方案（策略）  


对抗搜索： 代价->效用  
先考虑一个智能体：     
最小化代价改为最大化分数/效用。  
状态价值：从本状态出发可能获得的最大最终效用，递归计算。    

对抗搜索搜索树：
你一层我一层。

假定完全理性：极小极大(minimax)搜索。我方取最大，敌方取最小。玩家轮流行动直到最终。
遇到min节点取min，遇到max节点取max。  
不想象对方可以下一个反常（非最优）策略，因为对手这样选择让我方优势会更大。(但我输出的不一定是这种情况下的最优解，因为对方可能不是理性的)    
      
时间复杂度：O(b**m)
空间复杂度：O(bm)
完整博弈树不可能解出来。
降低复杂度：  

一，alpha-beta剪枝（以min节点为例）： 

一般流程(MIN 版本)
• 假设我们正在计算节点n的MIN
• 我们需要遍历n的子节点
• 随着进一步遍历，由于n取子节点最小值，因此n对于子节点最小值的估计会不断下降  
• 上层MAX会在意n节点的价值。  
• 设a是n到达根节点路径上各种MAX可以得
到的最大值  
• 如果n变得比a更差，MAX将避免它，从而
可以忽略n的其他子节点    
（手动画图思考）  
算法实现：
α: MAX’s best option  to root   
β: MIN’s best option  to root  
  
```python
#入口函数
def value(state , α, β):
if the state is a terminal state: 
    return the state’s utility
if the next agent is MAX: 
    return max-value(state , α, β)
if the next agent is MIN: 
    return min-value(state , α, β)
```

```python
def min-value(state , α, β):
initialize v = +∞
for each successor of state:
v = min(v, value(successor, α, β))
if v ≤ α :
    return v
β = min(β, v)
return v
```


```python
def max-value(state, α, β):
initialize v = -∞
for each successor of state:
v = max(v, value(successor, α, β))
if v ≥ β :
    return v
α= max(α, v)
return v

```

注意：
1.剪枝方式不会影响根节点值。
2.中间节点可能是错的。
3,最简单的版本无法进行后续的动作选择。

 二，搜索资源限制：  
有限深度搜索  
对于非叶子节点，将终态效用变为估值函数。  
  
估值函数：理想情况是返回实际的极大极小值（但不可实现）

三，其他博弈类型：多player或非零和博弈。  
极小极大搜索的推广：max-max-max tree  
终止状态给出效用元组，节点的价值也是效用元组。每个玩家最大化自己的部分，可以动态合作竞争。  

四，不确定的结果：
结果由偶然性控制，而不是对手。
最差情况vs平均情况。
当对手不是完美理性时，将min节点换成期望节点。（最坏情况过于保守，平均情况收益更大）  

期望最大化搜索：计算最优游戏过程下的平均得分。  
max节点与minimax相同，概率节点类似min但是结果不确定。计算预期结果（对其子节点加权求和）
