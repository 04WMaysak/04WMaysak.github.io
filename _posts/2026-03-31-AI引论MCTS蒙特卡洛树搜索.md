# 蒙特卡洛搜索


蒙特卡洛方法：  大规模随机抽样来近似问题的解  
随机试验次数越多越准确   
估计准确度与样本方差有关  

### 蒙特卡洛方法：围棋

例子：从当前开始四次模拟：黑棋赢了两次，白棋赢了两次，估值就是2/4=0.5  

产生问题：我们真的可以大规模抽样吗？
但是每一个状态只有很少的动作是高价值的

### 如何平衡抽样的不确定性

以老虎机为例：  
三台机器，不知道哪一台实际收益率最高。  
 矛盾点：既想抓住最优的玩下去，又怕自己判断错


$$
Q_{t+1}(a) = \begin{cases} \frac{1}{N_t(a)} \sum_{\{\tau | \tau \le t, A_\tau = a\}} R_\tau & , N_t(a) > 0 \\ 0 & , N_t(a) = 0 \end{cases}
$$

* **$k$**：动作的数量（比如，有多少老虎机）
* **$q_*(a)$**：动作 $a$ 的真实价值
* **$Q_t(a)$**：在时间 $t$ 的时候，对动作 $a$ 的估值
* **$N_t(a)$**：在时间 $t$ 及之前，动作 $a$ 被选中的次数
* **$R_t$**：在时间 $t$ 得到的价值/奖励
* **$A_t$**：在时间 $t$ 采取的动作
(好丑陋的符号)  
（其实就是通过目前收益和总尝试计算的单次收益期望）  

为了不漏掉最佳的老虎机，采取策略：
####  $\epsilon$ -greedy策略： 

给定一个概率 $\epsilon$ ：
* 抽取到 $1 - \epsilon$ 的时候：选当前最优的选择
* 抽取到 $\epsilon$ 的时候：随机选取一个当前不是最优的  

###### 注意
 探索是永远需要的，因为我们的估计始终不确定  
利用只是针对当前看起来最优的，别的
动作有可能更好  
• ε-greedy强制非最优动作也会被尝试，
但对所有非最优动作一视同仁。   
（采取到最优的概率在足够多的尝试后趋近于1-ε）  

• 如果更多尝试那些更有希
望成为更优的动作，效率将更高     

因此：
#### 最大置信（UpperConfidence Bound）(UCB)
* 已知：尝试次数**越多估值越准确**，也即我们对尝试次数少的估计信任不足（需要探索）
* 算法应平衡 **估值大小** 和 **次数多少**

$$
A_{t+1} = \operatorname{argmax}_a \left[ Q_t(a) + c \sqrt{\frac{\ln t}{N_t(a)}} \right]
$$

$c$：超参数（即练习前被人类定死的参数），用于平衡估值和信任度对决策的影响  
$$ lnt$$：测试次数越多，测试结果越准确，越不倾向于尝试。
（argmax意为取所有a中分数最大值）  


### 从UCB到博弈（单次动作到序列动作）

• 构建向前看的博弈树  
• 通过随机模拟，产生节点的估值函数  
• 在博弈树的中间节点，根据UCB来选择动作(这个动作的胜率+继续探索寻找神之一手)  

UCB:
$$Score = \underbrace{Q(\text{胜率})}_{\text{我想赢}} + \underbrace{c \sqrt{\frac{\ln T}{n}}}_{\text{看看去的少的地方}}$$
（没看过的新节点需要单独处理，可以说无穷大（强制探索），也可以是个大常数或者启发）    

在模拟中  
• 根节点：我方面临的情形   
• 根节点扩展出的路径：我方的不同选择  
• 按照深度不同，立场交替  
• 即：假设对手也选择较优策略  
##### 选择----->扩展------>模拟----->回溯（依次循环）

###### 选择
UCB Applied to Trees(UCT)：$$Score = \frac{w_i}{n_i} + c \sqrt{\frac{\ln N}{n_i}}$$
${w_i}$: 胜利次数
${n_i}$：子节点总尝试次数

如果它还有子节点，仍以此类推。  

###### 扩展
如果选择了新节点，进入扩展：  
每次只扩展一个节点。  

###### 模拟
碰到终点：返回胜负。    
没碰到：在这个节点进行一次蒙特卡洛模拟（扔骰子。也可以是加权随机），看胜负。  

（注：模拟经常不会选择一直运行到博弈
结束。而是会有一个最大步数，
然后直接返回启发值。）  
###### 回溯：  
把这个扔骰子的结果一步一步往回传，记录到每一个沿途节点上。直到根节点。  
（结果被存储，UCB值会被改变。接下来再次选择）
（蒙特卡洛树搜索需要一个终止条件，要不然会一直干下去）  

##### 一些其他特性：
蒙特卡洛树是不对称的

```python
import math
import random

class MCTS:
    def __init__(self, root_state):
        self.root = Node(state=root_state)

    def search(self, timeout):
        """主程序：在限定时间内不断迭代"""
        while not timeout():
            # 1. Selection & Expansion (TreePolicy)
            # 寻找一个还没试全动作的叶子节点，或者已经结束的节点
            leaf_node, reward_so_far = self.tree_policy(self.root)
            
            # 2. Simulation (DefaultPolicy / Rollout)
            # 从叶子节点开始闭眼随机下，直到分出胜负
            total_reward = self.default_policy(leaf_node.state, reward_so_far)
            
            # 3. Backpropagation
            # 把结果沿着父节点一路传回根部
            self.backpropagate(leaf_node, total_reward)
            
        return self.best_action(self.root)

    def tree_policy(self, node):
        """选择策略：负责在已有的树结构中找路，并适时扩展新节点"""
        reward = 0
        curr = node
        
        while not curr.state.is_terminal():
            # 如果当前状态下还有动作没被试过 (即子节点还没建全)
            if not curr.is_fully_expanded():
                # 扩展出一个新节点并立即返回，进入模拟阶段
                new_node = curr.expand()
                reward += curr.state.get_immediate_reward() 
                return new_node, reward
            else:
                # 如果动作都试过了，就用 UCB1 公式选一个最好的子节点继续下钻
                curr = curr.best_child_ucb()
                reward += curr.state.get_immediate_reward()
                
        return curr, reward

    def default_policy(self, state, current_reward):
        """仿真策略：纯随机模拟，不建立新节点"""
        curr_state = state
        reward = current_reward
        
        while not curr_state.is_terminal():
            action = random.choice(curr_state.get_legal_actions())
            next_state = curr_state.transition(action)
            reward += next_state.get_immediate_reward()
            curr_state = next_state
            
        return reward

    def backpropagate(self, node, reward):
        """反向传播：更新路径上所有节点的统计信息"""
        curr = node
        while curr is not None:
            curr.count += 1
            curr.reward += reward
            curr = curr.parent
```

 Algorithm 2: UCT 动作选择逻辑实.( SELECTACTION 过程)

```Python
import math
def select_action(position):
    """
    Algorithm 2: UCT Action Selection
    position 代表当前节点，包含其所有动作的选择统计
    """
    # 2: nActions ← |A|
    actions = position.get_legal_actions()
    n_actions = len(actions)
    
    # 3: nsum ← 0
    n_sum = 0
    # 4: for i ∈ {1..nActions} do
    for a in actions:
        # 5: nsum ← nsum + position.actions[i].n (累加动作被选中的总次数)
        n_sum += position.action_stats[a].n
    
    max_v = -float('inf')
    selected_action_index = -1
    
    # 8: for i ∈ {1..nActions} do
    for i, a in enumerate(actions):
        stats = position.action_stats[a]
        
        # 9: if position.actions[i].n = 0 then
        if stats.n == 0:
            # 10: v ← +∞ (未探索过的动作优先级最高)
            v = float('inf')
        else:
            # 12: v ← position.actions[i].value + sqrt(2*ln(nsum)/n)
            # 这里使用了图片中给定的特定 c = sqrt(2)
            exploitation = stats.value
            exploration = math.sqrt((2 * math.log(n_sum)) / stats.n)
            v = exploitation + exploration
            
        # 14: if v > maxv then
        if v > max_v:
            max_v = v
            # 16: j ← i
            selected_action_index = i
            
    # 19: return position.actions[j]
    return actions[selected_action_index]
```
###### 代码要点说明：
分母为 0 的处理：在 Algorithm 2 的第 9-10 行，代码明确处理了 $n=0$ 的情况，将其设为 inf。这确保了算法会先“横向铺满”所有没试过的动作，再深挖。

单子节点扩展：Algorithm 1 的 tree_policy 中，只要 if 发现一个 null 子节点，就会立即 return 并在该点开始 Simulation，符合“每一轮只扩展一个节点”的原则。

中间奖励与终局奖励：代码中的 r += reward_function(...) 允许游戏在过程中产生分数，而不仅仅是最后定胜负。    

UCT 公式：Algorithm 2 第 12 行实现了公式 $A_{t+1} = \text{argmax}_a \left[ Q_t(a) + c \sqrt{\frac{\ln t}{N_t(a)}} \right]$，其中图片 2 的伪代码将 $c$ 设为了 $\sqrt{2}$。
