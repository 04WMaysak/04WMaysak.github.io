C++类的继承，派生，复合  
<!--more-->  

为什么继承？——共同属性和方法（由此派生）+ 不同属性和方法  
特性：派生对象大小 = 基类对象的大小 + 派生类自己的成员变量的大小。  
（派生类对象包含基类对象，且基类对象保存在新增成员变量前）  

派生：基类/父类→派生类/子类  
派生：扩充与修改。可以添加新的成员，可以重写继承的成员。  
继承写法:  

class newclass : 派生方式说明符 base_class { … }  
例子：  
```cpp
class CUndergraduateStudent : public CStudent {  
private:  
    int nDepartment;  
public:  
    bool IsThreeGood() { ... }   // 覆盖  
    bool CanBaoYan() { ... }  
};  
```    
派生类拥有基类的全部成员，包括：  
•	private/protected/public成员变量  
•	private/protected/public方法  


派生类不能访问基类的private成员!!  
派生方式说明符：private / public / protected    
public继承：基类的成员在派生类中权限相同，但private不能访问。（c++的protected主要就是为继承设计的）    

protected继承：基类的public和protected成员在派生类中变为protected。  

private继承：基类的public和protected成员在派生类中都变为private。  


派生类的构造与析构：先调用基类，再调派生类构造函数。 
销毁时，先析构派生，再基类。（编译器自动调用基类，小心双重释放）  

派生类在基类无默认or缺省构造时必须在初始化列表中显式调用基类构造函数。有默认or缺省构造时可以不管。  
```cpp
class Base {
public:
    Base(int x) { ... }     // 只有带参构造函数，无默认构造
};

class Derived : public Base {
public:
    Derived() : Base(0) { ... }   // 必须显式调用
};
```

派生类定义基类同名的成员：覆盖。派生类访问时缺省访问派生类中定义的成员，  
访问基类同名成员，使用作用域符号 ::（如 Base::j = 1;）（备注：一般不覆盖变量，只覆盖函数）



复合对象：  
```cpp  
class CCircle {  
    double r;  
    CPoint center;  //另一对象作为成员。复合：“A有一个B”。继承：“A是一个B”
};  
```


如何避免循环定义：  
```cpp  
class CDog;                     // 前置声明    
class CMaster {      
    CDog * dogs[10];            // 编译器希望知道这个类有多大，但类的大小只有定义后才知道，指针的大小确定，因此使用指针可以规避问题。    
};    
class CDog {    
    CMaster * m;                // 使用指针    
};  
```
