# C++ 新特性核心笔记

## 1. 统一的初始化方法与成员变量默认值
* **统一初始化**：可以使用 `{ }` 对各种类型进行统一初始化。
    ```cpp
    int arr[3]{1, 2, 3};
    vector<int> iv{1, 2, 3};
    map<int, string> mp{{1, "a"}, {2, "b"}};
    string str{"Hello World"};
    int* p = new int[20]{1, 2, 3};
    struct A { int i, j; };
    A a1{10, 20}; // 自动调用构造函数
    ```
* **成员变量默认初始值**：在类定义时可直接为成员变量赋初值。
    ```cpp
    class B {
    public:
        int m = 1234; // 默认初始值
        int n;
    };
    ```

## 2. 类型推导关键字 (auto & decltype)
* **auto**：编译期自动判断变量类型。常用于较长的类型名（如迭代器）。
    ```cpp
    auto i = 100;           // int
    auto p = new A();       // A*
    map<int, int>::iterator it = mp.begin(); // 传统写法
    auto it = mp.begin();                    // auto 简化写法
    ```
    *结合返回值后置语法*：`auto add(T1 x, T2 y) -> decltype(x + y)`
* **decltype**：推导表达式的类型，但不实际计算表达式。
    ```cpp
    struct A { double x; };
    const A* a = new A();
    decltype(a) x1;       // A*
    decltype(a->x) x3;    // double
    decltype((a->x)) x4 = t; // double& (加括号表示引用)
    ```

## 3. 智能指针与空指针
需要包含头文件 `<memory>`。
shared_ptr：允许多个指针托管同一个对象。系统维护托管计数，计数归零时自动 `delete`。
    * *注意*：不能直接托管动态分配的数组（除非自定义删除器）。
    ```cpp
    shared_ptr<A> sp1(new A(2));
    shared_ptr<A> sp2(sp1);      // sp1 和 sp2 共同托管，计数为 2
    sp1.reset();                 // sp1 放弃托管，计数减 1
    A* p = sp2.get();            // 获取原始指针
    ```
* **unique_ptr**：独占对象的管理权（更快、更安全）。不可复制，但可以转移所有权。
    ```cpp
    unique_ptr<int> p1(new int(100));
    // unique_ptr<int> p2 = p1;  // 错误：不可复制
    unique_ptr<int> p2 = move(p1); // 正确：转移所有权，p1 置空
    ```
* **nullptr**：专门表示空指针，替代有歧义的 `NULL`（NULL 本质通常是整数 0）。

## 4. 基于范围的 for 循环
能够更简便地遍历数组或容器。
```cpp
int ary[] = {1, 2, 3, 4, 5};
for (int& e : ary) e *= 10;   // 引用遍历，可修改原值
for (int e : ary) cout << e;  // 值遍历，仅读取
for (auto& it : st) {}        // 搭配 auto 使用更加灵活
```

## 5. 右值引用与 move 语义
* **左值与右值**：能取地址的表达式是左值，不能取地址的（如临时对象、匿名变量 `A()`）是右值。
* **右值引用 (`&&`)**：专门绑定到右值上。
    ```cpp
    A&& r = A(); // 正确：绑定到右值
    ```
* **move 语义**：用于将资源（如动态内存）的所有权从一个对象转移到另一个对象，避免深拷贝，提高效率。
    ```cpp
    // 移动构造函数示例
    String(String&& s) : str(s.str) { 
        s.str = nullptr; // 剥夺原对象的资源
    }
    // std::move 将左值强制转换为右值，以触发移动构造/赋值
    s1 = move(s2); 
    ```
* **返回值优化**：
    * 返回局部对象：优先调用移动构造，如果没有则调用拷贝构造。
    * 返回全局对象：默认调用拷贝构造（若想移动需使用 `return move(全局对象)`）。

## 6. 无序容器 (哈希表)
* **unordered_map**：底层由哈希表实现，插入和查询的平均时间复杂度几乎是常数 $O(1)$。
    ```cpp
    #include <unordered_map>
    unordered_map<string, int> mp;
    mp.insert(make_pair("Dijkstra", 1972));
    mp["Ritchie"] = 1983;
    auto p = mp.find("Scott");
    ```

## 7. 正则表达式
需要包含头文件 `<regex>`。
```cpp
regex reg("b.?p.*k");
bool match = regex_match("bopggk", reg); // 返回 true 或 false
```

## 8. Lambda 表达式
* **语法**：`[捕获列表](参数列表) -> 返回值类型 { 函数体 }`（返回值类型可省略，由编译器推导）。
* **捕获列表规则**：
    * `[]`：不使用外部变量。
    * `[=]`：以传值形式捕获所有外部变量。
    * `[&]`：以引用形式捕获所有外部变量。
    * `[x, &y]`：`x` 传值，`y` 引用。
    * `[=, &x]` / `[&, x]`：混合模式。
* **本质**：Lambda 是一个匿名的函数对象（仿函数）。编译器会将其转化为一个包含重载了 `operator()` 的类（`struct`）。捕获的变量相当于类的成员变量。
    ```cpp
    int x = 10;
    auto f = [x](int a) { return a + x; };
    cout << f(5); // 输出 15
    ```

## 9. 模板类型参数包与递归 (可变参数模板)
允许模板接受任意数量、任意类型的参数。通常通过**递归拆包**的方式处理。
```cpp
template<typename... Ts> struct Tuple; // 声明参数包

// 递归特化：拆出第一个类型 T，剩下的 Ts 继续递归
template<typename T, typename... Ts>
struct Tuple<T, Ts...> {
    T value;
    Tuple<Ts...> rest;
    Tuple(T v, Ts... rest_args) : value(v), rest(rest_args...) {}
};

// 递归终止条件：空参数包
template<> struct Tuple<> {};

Tuple<int, double, string> t{10, 3.14, "hello"};
```

## 10. 万能引用 (Universal Reference) 与完美转发 (Perfect Forwarding)
* **万能引用 (`T&&`)**：在**函数模板**中，`T&&` 既能接收左值，也能接收右值。
* **完美转发 (`std::forward<T>`)**：在将参数传递给内部函数时，保持参数原本的值类别（左值仍为左值，右值仍为右值）。如果不使用 `forward`，命名的右值引用在函数内部会被当作左值处理。
    ```cpp
    template<typename T>
    void wrapper(T&& arg) { 
        // 自动将 arg 适配为对应的左值引用或右值引用版本
        process(std::forward<T>(arg)); 
    }
    ```

## 11. 多线程
需要包含头文件 `<thread>`。通过 `std::thread` 创建并执行线程。
```cpp
#include <thread>
void my_thread(int x) { /* ... */ }

int main() {
    thread th1(my_thread, 100); // 传入函数和参数
    
    // 或者使用可调用的对象/Lambda
    thread th2([]{ cout << "Thread 2\n"; });
    
    th1.join(); // 等待线程结束 (视业务需求而定)
    th2.join();
    return 0;
}
```
