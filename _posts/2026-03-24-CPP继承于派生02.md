C++类的继承，派生，复合  
<!--more-->  

课前小测：
派生类中重写基类同名同参函数：覆盖。 体现了派生类对继承得到的特点的修改    
（也可以通过BASE::function()来调用被覆盖的函数 ）  


访问权限说明符: protected  
基类的private:可以被下列函数访问  
◼基类成员  
◼基类友员  
基类的 private 成员在派生类中不能视为子类的成员，虽然的确存在。
  
public:可以被下列函数访问  
◼基类成员    
◼基类友员    
◼派生类成员  
◼派生类友员  
◼其他函数  


protected 成员的访问权限（介于 private 和 public 之间）
被声明为 protected 的成员，可以被以下三种角色访问：  
一、本类内  
成员函数  
友元函数  

二、派生类内部  
派生类的成员函数，可以访问当前对象（即this所指向）从基类继承的 protected 成员  
其他同类对象（同属一个派生类）从基类继承来的 protected 成员  


三、特殊注意：  
派生类的友元不能直接访问独立基类对象的 protected 成员（友元权限只针对完整的派生类本身,权限实质上和派生类成员函数一致，只不过没有this）  



```cpp
class  Father {
private: int nPrivate;          
public: int nPublic;            
//私有成员
//公有成员
protected: int nProtected;   // 保护成员
};
class Son : public Father{
  void AccessFather () {
    nPublic = 1;       // OK
    nPrivate = 1;     // wrong 
    nProtected = 1; // OK, 访问从基类继承的protected成
    Son s;
    s.nProtected = 1; //OK, 访问其它同类对象的基类保护成员
    Father f;
    f.nProtected = 1; //wrong, f不是当前对象，也不是Son的同类
}    
};

```

注：可以认为从public继承基类的子类内部有四种权限：  
基类 private —— 派生类不可直接访问。  
派生类自己的 private  —— 仅自身的成员函数和自身友元访问。  
基类+ 自己的 protected  —— 都可以在派生类内部（及进一步派生类中）访问。  
基类+自己的 public —— 任何地方都能访问。  


一个类访问另一个类的非公有成员的三种情况：
1. 同类。	同一个类的不同对象之间可以互相访问 private/protected 成员（通过对象名。
  
2. 友元	如果类 A 声明类 B 为友元（或某个函数为友元），则 B 可以访问 A 的所有成员。
    
3. 继承	派生类的成员函数可以访问本派生类对象中从基类继承来的 protected 成员，但不能直接访问独立的基类对象。



/这块有点乱//////////////



| 继承方式 | public 成员在派生类中权限 | protected 成员在派生类中权限 | private 成员在派生类中权限 |
|----------|---------------------------|-----------------------------|---------------------------|
| 公有继承 | public                    | protected                   | 不可见                    |
| 私有继承 | private                   | private                     | 不可见                    |
| 保护继承 | protected                 | protected                   | 不可见                    |


派生类的构造和析构：
创建派生对象：需要调用基类构造

◼ 从基类继承的成员
◼ 先执行基类的构造，再执行派生类构造。

调用基类构造函数:  
1◼ 显式: 在派生类的构造函数中, 为基类构造函数提供参数
```cpp
Derived(int a) : Base(a) {   // ✅ 正确
    //不能在函数体里面
}

```

2◼ 隐式方式: 省略基类构造时, 派生类的构造函数自动调用基类的默认构造。（提供了缺省构造，可以认为是默认构造函数）  
这两条对类内有其他成员对象的情况也适用。

析构：  
基类比派生类先构造，后析构。  
执行完派生类的析构后, 自动调用基类的析构函数  

总而言之，创建派生类的对象时,  执行一个派生类的构造函数之前:
◼ 基类构造  
◼ 成员对象类构造  

派生类析构时,  执行完派生类析构后:
◼ 成员对象类析构  
◼ 基类析构  

析构与构造调用顺序相反  


public 赋值兼容：  
只在 public继承下，派生类对象可以自动转换为基类对象，即赋值兼容：  

1 把派生类对象赋值给基类对象  
```cpp
b = d;
```
只复制基类部分，派生新增成员被切掉。即发生切片。

2 用派生类对象初始化基类引用
```cpp
base &br = d;//
```
引用绑定到派生类对象的基类子部分，可通过引用访问基类成员。

3 把派生类对象的地址赋给基类指针
```
base *pb = &d;//*pb看作base类，使用->访问基类public,不能访问派生类特有成员.
```

指针指向派生类对象的基类子对象，可通过指针访问基类成员（若有多态，可调用虚函数实现动态绑定）。
注意：赋值时发生“切片”，引用和指针保留多态特性。

通过强制指针类型转换,可以把ptrBase转换成Derived类的指针
```cpp
class Base{};
class Derived: public Base{
};
Base * ptrBase = &objDerived;
Derived *ptrDerived = (Derived * ) ptrBase;
```
◼ 程序员要保证ptrBase指向的是一个Derived！

向上转换（派生类→基类）：隐式转换，始终安全。ptrBase 指向 objDerived的基类子对象。

向下转换（基类→派生类）：需要显式转换。


间接基类： 
在声明派生类时,派生类的首部只需列出直接基类  
◼不要列出它的间接基类  
◼ 派生类沿着类的层次自动向上继承间接基类  
◼ 派生类的成员包括  
• 派生类成员  
• 直接基类成员  
• 间接基类的全部成员  


总 结
基本概念: 继承,基类,派生类
◼ 合理派生
派生类的成员组成,可见性
◼ private/protected成员的继承性
◼ 派生类的成员函数—不能访问基类中的private成员
◼ 派生类的成员函数—访问基类的protected成员
派生类的构造,析构
◼ 构造顺序: 基类,对象成员,派生
◼ 析构反之
派生类与基类的指针类型转换
◼ f(派生类)→y(基类对象): f: 对象/对象地址; y: 对象/引用/指针
◼ 基类指针→(强制指针类型转换)派生类指针



----考试范围外内容：多继承

1. 多重继承定义
一个类可以继承多个基类：

```cpp
class derived : access-specifier1 base1, access-specifier2 base2, … {
    // …
};
```
访问限定符（private、protected、public）分别控制每个基类的继承方式。

2. 构造函数调用顺序
顺序：按继承顺序依次调用各基类构造函数 → 再按成员声明顺序调用成员对象构造函数（若有） → 最后执行派生类构造函数体。

示例：若 Derived 继承自 Base1 和 Base2，则先调 Base1 构造，再调 Base2 构造。

3. 多继承中的重复基类（菱形继承）
若多个基类源于同一个公共基类，则最终派生类中会包含多份公共基类子对象（如 Base1 和 Base2 都继承自 Base，则 Derived 中有两个 Base 子对象），导致基类构造和析构多次调用。
解决方法：使用虚继承（virtual）。

虚继承：
普通继承按路径复制基类，虚继承按共享合并基类。
| 对比项 | 普通继承 | 虚继承 |
|--------|----------|--------|
| 基类子对象数量 | 在菱形继承中，最终派生类包含多份基类子对象 | 保证最终派生类中只包含一份虚基类子对象 |
| 构造调用次数 | 基类构造函数按继承路径调用多次 | 虚基类构造函数只调用一次（由最底层派生类负责） |
| 内存布局 | 基类成员直接连续存储 | 通过指针（或偏移量）间接访问虚基类成员，略有额外开销 |
| 初始化责任 | 每一层派生类都可初始化其直接基类 | 虚基类由最底层派生类在初始化列表中直接初始化（中间类对虚基类的初始化会被忽略） |
| 使用场景 | 常规继承关系 | 解决菱形继承中的二义性和数据冗余 |

| 转换方式 | 普通继承（单/多继承） | 虚继承（菱形继承） |
|----------|----------------------|-------------------|
| `static_cast` / C 风格强转 | 若基类指针确实指向派生类对象，通常安全（基类子对象在开头） | ❌ 不安全（基类子对象可能不在开头） |
| `dynamic_cast` | 安全（需基类有多态） | ✅ 安全（需基类有多态） |

基类有多态：该类至少包含一个虚函数（通常是虚析构函数或任何其他虚函数）。这样的类被称为多态类型，编译器会为其生成运行时类型信息（RTTI），从而支持 dynamic_cast 和 typeid 等操作。






1. 二义性问题
当多个基类包含同名成员（函数或变量）时，直接访问会产生二义性。

二义性检查优先于访问权限检查：即使某个基类的同名成员是私有的，只要存在两个同名成员，直接访问就是错误的。

解决：通过作用域解析符明确指定基类：
```cpp
d.base1::set(5);   // 调用 base1 中的 set
d.base2::set(5);   // 调用 base2 中的 set
```
5. 关键点
多重继承使派生类能组合多个基类的功能，但也可能带来复杂性。

构造函数调用顺序固定，但需注意重复基类问题。

二义性必须在编译时明确消除，不能依赖访问权限自动解决。
