课前回顾：  

封闭类  
嵌套了类对象作为成员。  
构造：先调用成员构造函数（按声明顺序而不是初始化列表！！！！），再执行封闭类自己的构造函数。要求进入封闭类构造函数体之前成员对象完全构造完毕。  
析构：先执行封闭类析构函数，再调用成员对象的析构函数（与声明顺序严格相反！！！
（该顺序是因为前后可能有依赖关系）。   


内联函数（inline）：编译器会在调用该函数的地方，直接插入函数体代码到调用点，不是执行常规调用）  
类的成员函数默认是内联的。  
（编译器视情况决定是否真的采取内联形式）



运算符重载实质上是函数重载。  
```cpp
返回值类型 operator 重载运算符(形式参数)
{
    .....
}//此处作为类成员函数有this指针，左操作数是本对象。
```


多次重载按照实参类型来确定调用哪一个函数。运算符重载成普通函数或类成员函数。  
多次重载由实参类型决定调用哪一个。  

静态(static ....)：  
静态成员函数：用类名调用，没有this指针。内部不能访问同类的非静态成员变量,也不能调用同类非静态成员函数

静态成员变量被所有对象共享（包括子类），没有对象存在,也可以访问类的静态成员。  
静态成员变量在类外初始化


友元： 一个类的友元函数/类中可以访问该类对象的protect,private成员  
友元类关系是单向的。  
在一个类中也可另一个类的成员函数声明为友元。    
友元关系不能传递（A友元的友元不是A的友元）  
也不会提供this指针。

本节主要讨论运算符重载的特殊情况：


1，赋值运算符‘=’只能重载为成员函数  

深浅复制：为什么重载=运算符  
浅复制：执行逐个字节的复制工作，两个指针指向同一个位置  
深复制：将一个对象中指针指向的内容复制过来。   
attention：自赋值问题。应当做特殊处理
```cpp
String & operator = (const String & s){
if( this == &s ) return * this;
delete [] str;//释放内存防止泄露
str = new char[strlen(s.str)+1];
strcpy(str, s.str);
return * this;
}
int main(){
String s;
s = "Hello";
s = s;
}//执行赋值之前内存已经释放，不做特殊处理则自赋值会导致选空指针
```

重载运算符的返回值：尽量保留运算符原本的特性
考虑:
```cpp 
a = b = c; 和 (a=b)=c;    //(a=b)表达式是a的引用
```
所以重载的=返回引用比较好。

对比复制构造与赋值：
复制构造时，目标对象还未诞生，不需要释放原有资源。
赋值时，目标对象已经拥有资源（比如动态分配的内存），必须先释放原有资源（防止内存泄漏），再拷贝新内容，还要防止自赋值。


可能用到：    
```cpp
#include <memory>
void* memcpy(void* 目标地址, const void* 源地址, size_t 字节数);
```  




重载为普通函数：不能访问private/protected！
重载为友元函数：
有些情况不希望左操作数是对象本身。
如
```cpp
C=5+C;
cout<<C;
```
例：  
```cpp
class Complex {
private:
    double real;
    double imag;

public:
    // 构造函数，默认值为0
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    // 声明为友元，以便访问私有成员
    friend ostream& operator<<(ostream& os, const Complex& c);
};

ostream& operator<<(ostream& os, const Complex& c) {
    os << c.real << " + " << c.imag << "i";
    return os;//当然函数也可以直接写在声明处
}

cout.operator << (n++).operator << (n);//其实也可以显式调用运算符，虽然很神经。
// 这个语句和cout<<n++<<n;是一样的。

```

重载[]:
```cpp
class  Array{
public:
Array(int n = 10) : size(n) {    ptr = new int[n];    }  
~Array()  {    delete [] ptr;    } 
int & operator[](int subscript){//通过引用来方便赋值操作
return ptr[subscript]; 
}
private:
int size;

};
```

```cpp
operator 目标类型();

return n; //不能加返回值类型！！！！
}

```
1,必须为成员函数,不指定返回类型,形参为空；
2,一般不改变被转换对象,因此常定义为const
3,类型转换自动调用


重载++/--
区分前置后置  
前缀：operator++()，无参数，返回引用（通常为T&），便于链式调用。
后缀自增：operator++(int)，带一个无用的int参数，返回副本（一般为原值)。(注意生命周期，此处直接return副本的引用不行)
```cpp

// 前置(一元。返回新值):
• T &operator++()    //成员函数
• T &operator--()
• T1 &operator++( T2 )    //全局函数
• T2 &operator--( T2 )
//++obj, obj.operator++() 或者operator++(obj) 都调用上述函数

//后置(二元。需要记录修改前的变量并且返回)
• T operator++(int)    //成员函数
• T operator--(int)
• T1 operator++(T2, int)    //全局函数
• T1 operator--(T2, int)
//obj++, obj.operator++(0)或者operator++(obj, 0) 都调用上述函数,后置传0以区分
```

运算符重载的注意事项  
  C++不允许定义新的运算符  
  重载后运算符的含义应该符合日常习惯  
  运算符重载不改变运算符的优先级  
  不能被重载：  “.”  “::”  “?:”  sizeof  
  重载运算符(),[],->或者赋值运算符=时,运算符重载，函数必须声明为类的成员函数。
