### 访问机制对比    
##### 1， 隐藏:    
所访问的对象由指针/引用类型决定    
用派生类指针访问, 访问派生类同名成员函数  
（静态绑定+名字隐藏）


用基类指针访问,访问的是基类同名成员函数,即使它指向的是一个派生类对象  

##### 2，虚函数（覆盖）:  
访问的对象由指针/引用实际指向的对象类型决定  

◼ 若该指针/引用 指向基类对象, 调用是基类的虚函数  
◼ 若指向派生类对象,调用派生类虚函数。  
（动态绑定）


##### 面向对象编程的三个基本概念：  
• 数据抽象-->用类进行数据抽象,封装  
• 继承-->派生类继承基类成员  
• 动态绑定-->多态,编译器在运行时决定使用基类or派生类中定义的函数  


### 动态联编  
函数调用在编译时无法确定调用哪个函数,运行到该语句才确定调用哪个函数。

1，基类指针/引用调用虚函数。
```cpp
class A {    public: virtual  void Get();   };
class B : public A { public: virtual  void  Get();  };
void MyFunction( A * pa ) {
pa->Get();
}
// pa->Get() 调用的是 A::Get()还是B::Get()?在编译时无法确定,因为不知道MyFunction被调用时,
// 形参会对应一个A 对象还是B对象
```
#### 虚函数在不同调用中的特殊行为
通过对象直接调用虚函数（而非指针/引用），编译器通常能确定对象类型，可能进行静态联编（但这不影响多态性）


2，构造和析构函数中的虚函数调用：在构造或析构期间，对象的动态类型被视为当前正在构造或析构的类，因此调用的虚函数版本是当前类定义的版本（而非最派生类），这是动态联编（绑定）的一个特殊场景

3，作用域运算符 :: 显式指定类名来调用虚函数时，会强制使用静态绑定 
  
（特别备注！！！联编和绑定是一回事！！）  

3,作用域运算符
```cpp
class Base {
public:
void fun1()  {    
fun2();    }
virtual void fun2()  { cout << "Base::fun2()" << endl;  }
};
class Derived : public Base {
public:
virtual void fun2() { cout << "Derived:fun2()" << endl; }
};
int main() {
Derived d;
Base * pBase = & d;
pBase->fun1();
return 0;
}//输出:Derived:fun2()
```

调用次序是：
```CPP
Base::fun1() -> Derived::fun2();
```
因为
```CPP
void fun1()  { 
fun2(); 
}
//相当于
void fun1()  { 
this->fun2();  
}
```
 编译时, 由于fun2()是虚函数, this是基类
指针,所以是动态联编
程序运行到fun1函数中时, this指针指向的是d, 所以经过动态联编,调用的是Derived::fun2()


“多态”的关键在于通过基类指针或引用调用虚函数时, 编译时不确定是基类还是派生类函数,运行时才确定。



### 多态的实现: 虚函数表
每个有虚函数的类(或有虚函数的类的派生类)都有一个虚函数表  
该类的任何对象中都放着虚函数表的指针  
虚函数表中列出了该类的虚函数地址     
（sizeof(对象)会比里面的成员之和更大）  

### 虚函数的访问权限：根据指针类型检查访问权限

```CPP
private:
virtual void fun2()  { cout << "Base::fun2()" << endl; }
};
class Derived:public Base {
public: 
virtual void fun2() { cout << "Derived:fun2()" << endl; }
};
Derived d;
Base * pBase = & d;
pBase-> fun2();  // 编译出错

```
 编译出错: fun2()是Base的私有成员. 即使实际调用 Derived的公有fun2()也不行。编译器语法检查是不考
虑运行结果！

如果将Base中的private换成public,即使Derived中的fun2()是private的, 编译依然能通过,也能正确调用Derived::fun2()

访问权限检查是根据指针类型来的


### 构造函数和析构函数中调用虚函数

 在构造函数和析构函数中调用虚函数时:
◼ 调用的函数是自己或基类中定义的函数,已经知道调哪个，不会等到运行时才决定。（静态绑定）

◼ 但在普通成员函数中调用虚函数, 则是动态联编, 是多态

```CPP
class myclass{
public:
virtual void hello(){  cout<<"hello from myclass"<<endl;  };
virtual void bye(){  cout<<"bye from myclass"<<endl;  };
};
class son : public myclass{
public:
void hello(){    cout<<"hello from son"<<endl;    };
son(){    hello();    };
~son(){    bye();    };
};
class grandson : public son{
public:
void hello()  {  cout<<"hello from grandson"<<endl; };
grandson()  {  cout<<"constructing grandson"<<endl; };
~grandson(){  cout<<"destructing grandson"<<endl; };
};
20
int main(){
grandson gson;
son *pson;
pson=&gson;
pson->hello();    //void grandson::hello()
return 0;
}
```
##### 输出结果:  
hello from son // gson先创建son, son()中静态连接了son::hello()
constructing grandson // gson的创建
hello from grandson    // pson->hello()动态连接到grandson::hello()
destructing grandson  // gson的创建的析构
bye from myclass
// gson中son部分的析构, ~son()中静态
//连接了myclass::bye()

### 虚析构函数

通过基类的指针删除派生类对象时:  
析构函数非虚，只调用基类的析构函数
◼但删除一个派生类的对象时,应当先调用派生类的析构函数,然后再调用基类的析构函数。

#### 解决方法：把基类的析构函数声明为virtual

◼派生类析构函数可以不加virtual
◼基类指针删除派生类对象时, 首调用派生类的析构函数,然后自动调用基类的析构函数。

一般来说, 一个类如果定义了虚函数, 则应
该将析构函数也定义成虚函数基类函数已是虚函数（包括析构）

#### 注意:
1,不允许以虚函数作为构造函数
2,基类函数已是虚函数（包括析构），是否重复写virtual无区别（自动继承虚属性）（但C++11以后推荐写上function(参数) override{}增加一层检查防错）  

3,作用域运算符 :: 显式指定类名来调用虚函数时，会强制使用静态绑定  

### 纯虚函数和抽象类
 包含纯虚函数的类叫抽象类

◼ 抽象类只能作为基类来派生新类使用,不能创建抽象类d对象
◼ 抽象类的指针和引用可指向由其派生出的类的对象


```CPP
A a ;                
// 错误, A 是抽象类,不能创建对象
A * pa ;           
// ok, 可以定义抽象类的指针和引用
pa = new A ;   //错误, A 是抽象类, 不能创建对象
```


• 抽象类的成员可以调用纯虚函数（这不会出错，思考一下）     
• 但是在构造函数或析构函数内部不能调用纯虚函数  
• 如果一个类从抽象类派生而来,那么当且仅当它实现
了基类中的所有纯虚函数,才成为非抽象类


```CPP
class A {
public:
virtual void f() = 0;
//纯虚函数
void g( ) {    this->f( ) ;    } //ok
A( ){   }                                
//如果是A(){  f( );  } 错误
};
class B : public A{
public:
void f(){    cout<<"B:f()"<<endl;     }
};
int main(){
B b;
b.g( );
return 0;
}
``` 
输出结果:  
B:f()
