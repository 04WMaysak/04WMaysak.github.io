```cpp

    virtual void Func() { cout << "A::Func" << endl; }

public:
    virtual void Func() { cout << "B::Func" << endl; }
};

int main() {
    A a;
    A * pa = new B();
    pa->Func();

    pa->Func();

    return 0;
}
```

## 模板

### 基本概念
函数模板是 C++ 泛型编程的基础。它让你的代码“与类型无关”，提高了代码的复用性  
编译系统根据函数调用时实参的类型, 自动生成相应的模板函数  


### 函数模板
一个函数模板可以有多个类型参数


```cpp
template<class T1, class T2>
//此处写class实际上是万能的，其实不需要传递类对象，为避免歧义也可以写typename
//template只作用于后面紧跟的定义,每一个独立的函数或类，都必须拥有自己专属的 template!!
//一般不会要求特定类型，如果要求会用特殊语法而不是class换成int等。
void print(T1 arg1, T2 arg2)
{    
    cout<<arg1<<"    "<<arg2<<endl;  return;    }

//函数模板的参数类型- 可以用类型参数(也就是T1,T2)说明- 也可以用基本数据类型
//其他的类说明
template<class T1, class T2>
void print(T1 arg1, T2 arg2, string s, int k)
{      
    cout<<arg1<<s<<arg2<<k<<endl; 
    return; 
}
```

#### 二义性问题
赋值兼容原则引起函数模板中类型参数的二义性

```cpp
template<class T>
T myFunction( T arg1, T arg2)
{ cout<<arg1<<“   ”<<arg2<<“\n”; return arg1;} …
myFunction(5, 7); //ok: replace T with int
myFunction(5.8, 8.4); //ok: replace T with double
myFunction(5, 8.4); //error: replace T with int or double? 二义性
```

可以在函数模板中使用多个类型参数,可以避免二义性

```cpp

template<class T1, class T2>
T1 myFunction( T1 arg1, T2 arg2)
{ cout<<arg1<<“   ”<<arg2<<“\n”; return arg1;} …
myFunction(5, 7); //ok: replace T1 and T2 with int
myFunction(5.8, 8.4); //ok: replace T1 and T2 with double
myFunction(5, 8.4); //ok: replace T1 with int, T2 with double

```

##### 匹配顺序

1. 先找参数完全匹配的函数  
2. 再找参数完全匹配的模板  
3. 在没有二义性的前提下,再找参数经过自动转换后能够匹配的函数  
4. 都找不到, 报错  


```cpp
template <class T> 
void Max( T a, T b ) {
cout << "TemplateMax" <<endl;
return;
}
void Max(double a, double b){
cout << "MyMax" << endl;
return;
}
int main(){
int i=4, j=5;
Max(1.2, 3.4); //输出MyMax
Max(i, j);        
//输出TemplateMax
Max(1.2, 3);   //强制类型转换3为double, 调用普通Max函数
}
/*
输出
MyMax
TemplateMax
MyMax
*/
```

```cpp
}

//调用第一个T Max(T a, T b)模板生成的函数

```


| 调用           | 选择原因                         | 输出              |
|----------------|----------------------------------|-------------------|
| Max(1.2, 3.4)  | 普通函数优先于模板               | MyMax             |
| Max(i, j)      | 模板1比模板2更特化               | Template Max 1    |
| Max(1.2, 3)    | 模板2精确匹配，普通函数需转换    | Template Max 2    |

### 类模板
为了快速定义出一批相似的类, 可以定义类模板  
由类模板→生成不同的类    
数组是一种常见的数据类型,元素可以是:- 整数，学生，字符串…  
考虑一个数组类,需要提供的基本操作：    
1. len(): 查看数组的长度  
2. getElement(int index): 获取其中的一个元素  
3. setElement(int index): 对其中的一个元素进行赋值    

这些数组类, 除了元素的类型不同之外, 其他的完全相同  
在定义类的时候给它参数, 这些参数表示类型。    
在调用类模板时, 指定参数, 由编译系统根据参数提供的类型自动产生相应模板类。  
  
只要模板参数相同，就是同一个类，它们static 变量也是同一个。

```cpp
template <class T>   //类模板的首部, 声明类模板的参数
class CArray{
T*ptrElement;
int size;
public:
CArray(int length);
~ CArray();
int len();
void setElement(T arg, int index);
TgetElement(int index);
};
```

类模板的定义
template:
• CArray是一个类模板
• 声明类型参数,用来定义CArray属性类型
.成员函数的参数类型和返回值类型

写法：

```cpp
C++的类模板的写法如下:
template <类型参数表>
class 类模板名
{
成员函数和成员变量
};
类型参数表:
class 类型参数1, class 类型参数2, …
```

#### 类模板与派生  

| 情况 | 基类 | 派生类 | 基类类型是否随派生类变化 |
|------|------|--------|--------------------------|
| (1) 模板→模板 | 类模板 | 类模板 | ✅ 是（用派生类的模板参数实例化） |
| (2) 模板类→模板 | 类模板的某个实例（固定） | 类模板 | ❌ 否（固定为 A<int,double> 等） |
| (3) 普通类 →模板 | 普通类 | 类模板 | ❌ 否（固定为 A） |
| (4) 模板类→普通类 | 类模板的某个实例（固定） | 普通类 | ❌ 否（固定为 A<int> 等） |

在继承中，无论如何基类模板的模板参数必须被指定        
怎么写：
（1）

```cpp
//1：模板到模板

class A {
T1 v1; T2 v2;
};
template <class T1,class T2>
class B: public A<T2,T1> {
T1 v3; T2 v4;
};

//2：具体模板类到模板
template <class T1, class T2>
class A {    
T1 v1; 
T2 v2;
};

template <class T>
class B: public A <int, double> { 
T v; 
};
int main()  {

//3：普通类派生出模板

//4，模板类派生出普通类。

};
```

#### 友元关系
  
| 类别 | 场景描述 | 友元范围与逻辑说明 |
| --- | --- | --- |
| 1 | 普通函数/类/成员函数作为类模板的友元 | 逻辑：所有的类模板实例都将该函数或类视为友元。 |
| 2 | 函数模板作为类模板的友元 |通常用于重载运算符。从函数模板生成的任何实例（任意 T, U）都是该类模板任何实例的友元。多对多的宽泛授权。 |
| 3 | 函数模板作为普通类的友元 |函数模板生成的函数,都成为类A 的友元 |
| 4 | 类模板作为类模板的友元 | 逻辑：类模板 A 的所有实例都是类模板 B 的友元。 A<任意类型> 能访问 B<任意类型> 的私有成员。 |
| 5 | 类型参数作为友元 | 逻辑：根据传入模板的具体类型 T 来决定友元。如果 CTemp<myClassA, 10>，函数内声明友元关系，则 myClassA 是该实例的友元。 |
  
示例：
```cpp
void Func1() {   } //函数
class A {   };
//类
class B{
public:
void Func() {   } //类的成员函数
};
template <class T>
class Tmpl {
friend void Func1();
friend class A;
friend void B::Func();
}; //任何从Tmpl实例化来的类,都有以上三个友元




#include <iostream>
#include <string>
using namespace std;
template <class T1, class T2>
class Pair{
T1 key;    
//关键字
T2 value;  //值
public:
Pair(T1 k, T2 v):key(k), value(v) { };

template <class T3, class T4>
friend ostream & operator<<( ostream & o, const
Pair<T3, T4> & p ); //函数模版
};

template <class T3, class T4>
ostream & operator<< (ostream & o, const Pair<T3, T4> & p){
o << "(" << p.key << "," << p.value << ")" ;
return o;

/*
任意从
template <class T3, class T4>
ostream & operator<< (ostream & o, const Pair<T3,T4> & p)
生成的函数,都是任意Pair 模板类的友元
*/

```

### 字符串流处理
除了标准流和文件流输入输出外, 还可以从string进
行输入输出  
类似istream和osteram进行标准流输入输出,  
用istringstream 和 ostringstream进行字符串上的
输入输出, 也称为内存输入输出  

把字符串当作输入源进行解析，以及用输出流方便地构造复杂格式的字符串。  

| 特性 | istringstream | ostringstream |
|------|---------------|----------------|
| 作用 | 从字符串中解析数据（类似 scanf 或 cin） | 将数据格式化为字符串（类似 printf 或 cout 但写入字符串） |
| 常用操作 | `>>` 提取 | `<<` 插入 |
| 获取/设置底层字符串 | 构造时传入，或通过 `str()` 读取/修改 | 通过 `str()` 获取结果，也可用 `str("...")` 重置 |
| 头文件 | `<sstream>` | `<sstream>` |




```cpp
#include <string>
#include <iostream>
#include <sstream>
// 例: 字符串输入流
string input("Input test 123 4.7 A");
istringstream inputString(input);
string string1, string2;
int i;
double d;
char c;
inputString >> string1 >> string2 >> i >> d >> c;
cout << string1 << endl << string2 << endl;
cout << i << endl << d << endl << c <<endl;

long l;
if(inputString >> l)    cout << "long\n";
else    
cout << "empty\n";
/*
输出:
Input
test
123
4.7
A
empty

*/

//例: 字符串输出流
string input("Output test 123 4.7 A");
istringstream inputString(input);
string string1, string2;
int i;
double d;
char c;
inputString >> string1 >> string2 >> i >> d >> c;
ostringstream outputString;
outputString << string1 << endl << string2 << endl;
outputString << i << endl << d << endl << c <<endl;
cout << outputString.str();

输出：
Output
test
123
4.7
A
```
