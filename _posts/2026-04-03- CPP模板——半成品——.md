
一段非常神人的展示虚表特性的代码

```cpp

class A {
public:
    virtual void Func() { cout << "A::Func" << endl; }
};

class B : public A {
public:
    virtual void Func() { cout << "B::Func" << endl; }
};

int main() {
    A a;
    A * pa = new B();
    pa->Func();

    // 32位的地址长度是4字节，也就是long 若是 64 位程序，虚表指针的大小（一个地址长度）为 8 字节，则应为 long long
    long * p1 = (long *) &a;
    long * p2 = (long *) pa;

    * p2 = * p1;//拷贝了p1的虚指针,现在p2的虚指针和p1一样了，因此实现p1的虚函数

    pa->Func();

    return 0;
}
//输出：
// B::func
// A::func
//不要在实际开发中这么写
```

## 模板

### 基本概念
函数模板是 C++ 泛型编程的基础。它让你的代码“与类型无关”，提高了代码的复用性  
编译系统根据函数调用时实参的类型, 自动生成相应的模板函数  


### 函数模板
一个函数模板可以有多个类型参数


```cpp
template<class T1, class T2>
//此处写class实际上是万能的，其实不需要传递类对象，为避免歧义也可以写typename
//template只作用于后面紧跟的定义,每一个独立的函数或类，都必须拥有自己专属的 template!!
//一般不会要求特定类型，如果要求会用特殊语法而不是class换成int等。
void print(T1 arg1, T2 arg2)
{    
    cout<<arg1<<"    "<<arg2<<endl;  return;    }

//函数模板的参数类型- 可以用类型参数(也就是T1,T2)说明- 也可以用基本数据类型
//其他的类说明
template<class T1, class T2>
void print(T1 arg1, T2 arg2, string s, int k)
{      
    cout<<arg1<<s<<arg2<<k<<endl; 
    return; 
}
```

#### 二义性问题
赋值兼容原则引起函数模板中类型参数的二义性

```cpp
template<class T>
T myFunction( T arg1, T arg2)
{ cout<<arg1<<“   ”<<arg2<<“\n”; return arg1;} …
myFunction(5, 7); //ok: replace T with int
myFunction(5.8, 8.4); //ok: replace T with double
myFunction(5, 8.4); //error: replace T with int or double? 二义性
```

可以在函数模板中使用多个类型参数,可以避免二义性

```cpp

template<class T1, class T2>
T1 myFunction( T1 arg1, T2 arg2)
{ cout<<arg1<<“   ”<<arg2<<“\n”; return arg1;} …
myFunction(5, 7); //ok: replace T1 and T2 with int
myFunction(5.8, 8.4); //ok: replace T1 and T2 with double
myFunction(5, 8.4); //ok: replace T1 with int, T2 with double

```


#### 模板函数的重载
函数模板与函数模板的重载: 同一函数名,参数的数量不同  
```cpp
template<class T>
T myFunction( T arg )
{ cout<< “one argument\n”; return arg;}
template<class T1, class T2>
T1 myFunction( T1 arg1, T2 arg2 )
{ cout<< “two arguments\n”; return arg1;}
…
myFunction(5); //ok: replace with int
myFunction(5.8, 8.4); //ok: replace T1 and T2 with double
myFunction(5, 8.4); //ok: replace T1 with int, T2 with double
```
##### 匹配顺序

1. 先找参数完全匹配的函数  
2. 再找参数完全匹配的模板  
3. 在没有二义性的前提下,再找参数经过自动转换后能够匹配的函数  
4. 都找不到, 报错  


```cpp
template <class T> 
void Max( T a, T b ) {
cout << "TemplateMax" <<endl;
return;
}
void Max(double a, double b){
cout << "MyMax" << endl;
return;
}
int main(){
int i=4, j=5;
Max(1.2, 3.4); //输出MyMax
Max(i, j);        
//输出TemplateMax
Max(1.2, 3);   //强制类型转换3为double, 调用普通Max函数
}
/*
输出
MyMax
TemplateMax
MyMax
*/
```

例: 函数模板调用顺序

```cpp
template <class T> 
void Max(T a, T b){
cout << "Template Max 1" <<endl;
return 0;
}
template <class T, class T2> 
void Max(T a, T2 b){
cout << "Template Max 2" <<endl;
return 0;
}

void Max(double a, double b){
cout << "MyMax" << endl;
return 0;
}
int main() 
{
int i=4, j=5;
Max(1.2, 3.4);  //调用Max(double, double)函数
Max(i, j);         
/*运行结果:
MyMax
Template Max 1
Template Max 2
*/
//调用第一个T Max(T a, T b)模板生成的函数
Max(1.2, 3);    //调用第二个T Max(T a, T2 b)模板生成的函数
return 0;
}
```


| 调用           | 选择原因                         | 输出              |
|----------------|----------------------------------|-------------------|
| Max(1.2, 3.4)  | 普通函数优先于模板               | MyMax             |
| Max(i, j)      | 模板1比模板2更特化               | Template Max 1    |
| Max(1.2, 3)    | 模板2精确匹配，普通函数需转换    | Template Max 2    |

### 类模板
为了快速定义出一批相似的类, 可以定义类模板
由类模板→生成不同的类  
数组是一种常见的数据类型,元素可以是:- 整数，学生，字符串…
考虑一个数组类,需要提供的基本操作：  
1. len(): 查看数组的长度
2. getElement(int index): 获取其中的一个元素
3. setElement(int index): 对其中的一个元素进行赋值  

这些数组类, 除了元素的类型不同之外, 其他的完全相同
在定义类的时候给它参数, 这些参数表示类型。  
在调用类模板时, 指定参数, 由编译系统根据参数提供的类型自动产生相应模板类。

```cpp
template <class T>   //类模板的首部, 声明类模板的参数
class CArray{
T*ptrElement;
int size;
public:
CArray(int length);
~ CArray();
int len();
void setElement(T arg, int index);
TgetElement(int index);
};
```

类模板的定义
template:
• CArray是一个类模板
• 声明类型参数,用来定义CArray属性类型
.成员函数的参数类型和返回值类型

写法：

```cpp
C++的类模板的写法如下:
template <类型参数表>
class 类模板名
{
成员函数和成员变量
};
类型参数表:
class 类型参数1, class 类型参数2, …
```

#### 类模板的定义

```

• 类模板里的成员函数,在类模板外定义时:
template <类型参数表>
返回值类型  类模板名<类型参数名列表>::成员函数名(参数表){ 
…
}

• 用类模板定义对象:   
类模板名<真实类型参数表>  对象名(构造函数实际参数表);  

• 有无参构造函数,可以只写:  
类模板名<真实类型参数表>  对象名;  
```

```cpp
Pair类模板:
template <class T1, class T2>
class Pair {
public:
T1 key;     //关键字
T2 value;  //值
Pair(T1 k, T2 v):key(k), value(v) { }
bool operator < ( const Pair<T1, T2> & p) const; 
};
template<class T1, class T2>
bool Pair<T1, T2>::operator < ( const Pair<T1, T2> & p) const 
//Pair的成员函数operator <
{    return key < p.key;    }
```

#### 函数模版作为类模板成员
```cpp
#include <iostream>
using namespace std;
template <class T>
class A{
public:
template <class T2>  
void Func(T2 t) {    cout << t;    }    //成员函数模板。注意此处使用T这一名字代替T2会报错
};
int main()  {
A<int> a;
a.Func(‘K’);           
//成员函数模板Func被实例化
a.Func(“Hello”);   //成员函数模板Func被实例化
return 0;
}
```
类模板中的成员函数可以是函数模板, 
该成员函数只有在被调用时才会实例化


#### 非类型模板参数
类模板的<类型参数表>中可以包括非类型参数(non-type parameter)
◼ 非类型参数:   
用来说明类模板中的属性  
◼ 类型参数:   
用来说明类模板中的属性类型, 成员函数的参数类型和
返回值类型  

可以提高执行效率  

```cpp
template <class T, int Size>
void printArray(const T (&arr)[Size]) {
    for (int i = 0; i < Size; ++i) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;
}
```
