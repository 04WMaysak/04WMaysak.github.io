虚函数  
在类定义中,前面有virtual 关键字的成员函数  

```cpp
class base {
virtual int get() ;
};
int base::get() { 
// …
}
```
• 1,virtual 关键字只用在类定义里的函数声明  
• 2,写函数体时不用(这一点同friend,static)
• 3,静态成员函数不能是虚函数  
针对第三点补充：虚函数依赖对象实例，而静态成员函数不依赖对象。  
1. 虚函数的工作原理  
虚函数通过虚函数表（vtable） 和虚指针（vptr） 实现动态绑定： 
每个含有虚函数的类都有一个虚函数表，里面存放着虚函数的地址。  

每个对象内部会有一个隐藏的虚指针（vptr），指向该类的虚函数表。  

当通过基类指针或引用调用虚函数时，程序会根据对象的实际类型，通过 vptr 找到正确的虚函数表，再调用对应的函数。  

因此，虚函数的调用依赖于对象，因为需要从对象的 vptr 获取信息。  

函数与基类虚函数签名不同，那么它不是覆盖，只是隐藏
（签名 = 函数名 + 参数类型列表 + const/引用限定符（返回类型不参与）

多态的表现形式-1  
1 派生类的指针可以赋给基类指针   
2 通过基类指针调用基类和派生类中的同名虚函数时:   
 若该指针指向一个基类的对象,调用基类的虚函数- 若该指针指向派生类的对象,则调用派生类的虚函数。这种机制是多态。  
即根据指针或引用（引用不可修改指向）指向的对象类型来决定调用的函数
```cpp
class base {
public:
	virtual void func() {//type— void或者type- virtual都行
		cout << "base virtual func\n";
	}
};
class derived :public base {
	void func() {//除了父类之外，同名函数可以不加void
		cout << "derived virtual func";
	}
};
int main()
	{
	base b;
	derived d;
	base* ptr = &b;//ptr在编译时确定的静态类型：base。这决定了可访问成员。函数调用时，编译器根据静态类型选择候选函数（重载解析）
	ptr->func();
	ptr = &d;//运行时的动态类型：derived.虚指针依此在虚函数表中寻找虚函数。
	ptr->func();
	/*输出：base virtual func
	derived virtual func
    */
    }
```

多态的表现形式-2  
 派生类的对象可以赋给基类引用  
 通过该基类引用调用基类和派生类中的同名虚函数时:- 若引用的是基类对象,调用基类虚函数；若引用派生类对象,调用派生类的虚函数。与指针一致。  
 
 例如：  
```cpp

r.SomeVirtualFunction();
```

多态-3
虚函数首先是一个普通成员函数，可以在类内定义，也可以通过对象直接调用。  
只有在通过指针或引用调用时，才触发动态绑定（多态）；直接通过对象调用时，与普通成员函数无异。  

多态的特点：
实质  
◼ 父类定义共同接口,子类不同实现  
◼ 通过父类以相同的方式操作不同子类的行为  

意义：  
◼ 能够增强可扩充性    
◼ 程序需要修改或增加功能的时候,改动代码较少

***补充知识：
using Base::Base_member;(解决基类同名函数在派生类中被隐藏的问题。可以直接调用成员而不需要Base::)

```cpp
class Base {
public:
    void func(int) {}
    void func(double) {}
};

class Derived : public Base {
public:
    using Base::func;   // 把基类的两个 func 引入
    void func(char) {}  // 新增重载
};
Derived d;
d.func(5);    // 调用 Base::func(int)
d.func(3.14); // 调用 Base::func(double)
d.func('a');  // 调用 Derived::func(char)
```
***补充知识：override
```cpp
void func() override { }  
//加入override可以让编译器检查虚函数覆盖的签名是否匹配，避免意外隐藏。
```

***补充知识：函数定义
必须声明：任何成员函数（包括虚函数覆盖）都需要在类内声明，否则无法被其他成员调用。  
声明方式：可以直接写函数体（类内定义），也可以只写原型（类外定义）。  

顺序无关：只要在类内声明了，定义在前在后不影响其他成员调用。  
 

 
***补充知识：重载解析。

重载解析：编译器根据静态类型和实参，在编译时从多个同名函数中选出最匹配的函数签名。

对于非虚函数：选出的签名就是最终调用的函数。

对于虚函数：选出的签名决定调用虚函数表中的哪个条目，运行时根据实际对象类型调用最终实现。

非虚函数：
```cpp
class Base {
public:
    void show(int x) { cout << "Base::show(int)\n"; }
    void show(double d) { cout << "Base::show(double)\n"; }
};

class Derived : public Base {
public:
    void show(int x) { cout << "Derived::show(int)\n"; }   // 隐藏基类两个 show
};

int main() {
    Derived d;
    Base* p = &d;          // 静态类型 Base*
    p->show(5);            // 输出 Base::show(int)
                           // 编译器只看 Base 中的重载，选 Base::show(int)
    d.show(5);             // 输出 Derived::show(int)
                           // 派生类自己的 show(int) 隐藏了基类版本
}
```
虚函数示例（重载解析 + 动态绑定）
```cpp
class Base {
public:
    virtual void func(int x) { cout << "Base::func(int)\n"; }
    virtual void func(double d) { cout << "Base::func(double)\n"; }
};

class Derived : public Base {
public:
    void func(int x) override { cout << "Derived::func(int)\n"; }   // 覆盖 int 版本
    // 未覆盖 double 版本
};

int main() {
    Derived d;
    Base* p = &d;          // 静态类型 Base*
    p->func(5);            // 重载解析选 Base::func(int) 签名
                           // 动态绑定到 Derived::func(int) → 输出 Derived::func(int)
    p->func(3.14);         // 重载解析选 Base::func(double)
                           // 派生类未覆盖 → 输出 Base::func(double)
}
```
为什么重载解析只看静态类型？
C++ 是静态类型语言，编译时只能根据声明的类型确定函数签名。对于虚函数，签名在编译时确定，动态绑定在运行时根据实际对象完成。
